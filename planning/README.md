# Planning

## Purpose
Project planning, status tracking, and session documentation.

## Contents
| File | Purpose |
|------|---------|
| STATUS.md | Current project state and next actions |
| sessions/ | Historical session logs (if needed) |

## For AI Agents

### Before Starting Work
1. Read `STATUS.md` to understand current state
2. Check what was done in previous sessions
3. Identify blockers or pending decisions

### After Completing Work
1. Update `STATUS.md` with:
   - What was accomplished
   - Current blockers (if any)
   - Next priority actions
2. Create session log if significant work was done

### STATUS.md Format
```markdown
# Project Status

## Current State
[1-2 sentences describing where the project is]

## Recent Changes
- [Date]: [What was done]

## Blockers
- [ ] [Blocking issue]

## Next Actions
1. [Most important next step]
2. [Second priority]
3. [Third priority]

## Notes
[Any context for future sessions]
```

### Session Logs
Only create if:
- Significant feature completed
- Major debugging session
- Architectural decisions made

Format: `sessions/YYYY-MM-DD-description.md`
